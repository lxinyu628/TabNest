## 二、目标愿景

> **定位**：极简信息流起始页，标签管理为核心差异化能力
>
> **设计哲学**：简约不简单，信息密度适中，交互流畅无广告
>
> **竞品差距**：不做 iTab 的"小组件堆砌"，走"精准克制"路线

## 三、完整功能规划

### 🔴 Phase 4 — 核心体验层

#### 4.1 天气小组件
**目标**：顶部显示实时天气 + 问候语增强

| 功能点 | 说明 |
|--------|------|
| 天气数据 | 调用高德/Open-Meteo 免费 API（无需 Key） |
| 展示内容 | 温度 + 天气图标 + 城市名 |
| 位置获取 | 浏览器 Geolocation API（用户授权） |
| 降级策略 | 无权限时显示默认城市 / 手动设置 |
| 缓存 | 本地缓存 30 分钟，避免频繁请求 |

**UI 位置**：topbar 左侧，时钟旁边

**API 选择**：Open-Meteo（完全免费，无需注册）
```
GET https://api.open-meteo.com/v1/forecast
  ?latitude=XX&longitude=XX
  &current_weather=true
  &timezone=auto
```

---

#### 4.2 极简模式（Zen Mode）
**目标**：一键切换纯时钟界面，无任何干扰元素

| 功能点 | 说明 |
|--------|------|
| 触发方式 | 点击时钟区域 / 快捷键 `Z` |
| 退出方式 | 任意键 / ESC / 点击任意位置 |
| 隐藏元素 | 搜索框、快捷书签、一言、底部栏 |
| 显示内容 | 全屏时钟（超大字体）+ 壁纸（若有） |
| 状态持久化 | 不保存，每次新标签页重置为正常模式 |

---

#### 4.3 一言/语录模块
**目标**：底部显示随机短句，增强人文感

| 功能点 | 说明 |
|--------|------|
| 数据源 | Hitokoto API（免费） |
| 展示位置 | 快捷书签下方，底部栏上方 |
| 内容类型 | 可选：诗词/励志/一言/自定义 |
| 刷新 | 每 30 秒自动换一条 / 点击手动刷新 |
| 关闭入口 | 设置面板中可关闭一言模块 |

---

### 🟡 Phase 5 — 内容丰富层

#### 5.1 壁纸系统
**目标**：支持背景图片/纯色渐变

| 功能点 | 说明 |
|--------|------|
| 内置壁纸 | 5-8 张精选渐变/风景图片（内置 base64） |
| 纯色渐变 | 预设 5 种渐变色方案（薄荷绿/珊瑚橙/薰衣草/深空灰/原生日出） |
| 本地上传 | 用户可上传本地图片作为壁纸 |
| 随机切换 | 每次打开新标签页随机切换 |
| 毛玻璃效果 | 背景图片时自动叠加毛玻璃（blur + 降饱和） |
| 存储 | `chrome.storage.local` 存储壁纸配置 |

**UI 入口**：设置面板 → 壁纸选择器（网格预览）

---

#### 5.2 快捷书签编辑
**目标**：用户可自行增删改快捷链接

| 功能点 | 说明 |
|--------|------|
| 编辑模式 | 设置面板中「编辑快捷书签」按钮 |
| 操作 | 添加 / 删除 / 拖拽排序 / 批量操作 |
| 数据结构 | 名称 + URL + 图标（emoji 或 favicon） |
| 图标获取 | 自动拉取网站 favicon（Google Favicon API） |
| 默认分组 | 社交 / 工作 / 工具 / 其他（可自定义分组名） |
| 导入书签 | 支持从浏览器书签 HTML 导入（后续） |
| 存储 | `chrome.storage.local` |

**UI 形态**：抽屉面板（右侧），编辑模式切换

---

#### 5.3 热搜小组件（可选）
**目标**：显示实时热搜榜单

| 功能点 | 说明 |
|--------|------|
| 数据源 | 微博热搜 / 知乎热榜 / 哔哩哔哩热榜（公开接口） |
| 展示数量 | 每次显示 5 条，点击展开全部 |
| 位置 | 快捷书签下方或侧边栏 |
| 刷新间隔 | 每 15 分钟 |
| 关闭入口 | 可在设置中关闭 |

> ⚠️ 注意：热搜数据源可能存在 CORS/API 限制，先评估可行性

## 四、推荐开发顺序

```
Phase 4（核心体验）
  ├─ 4.1 天气小组件        ← 价值感知最高
  ├─ 4.2 极简模式          ← 技术实现简单，交互感强
  └─ 4.3 一言模块          ← 内容丰富，即时可用

Phase 5（内容丰富）
  ├─ 5.1 壁纸系统          ← 视觉效果提升显著
  ├─ 5.2 快捷书签编辑      ← 核心自定义能力
  └─ 5.3 热搜小组件        ← 可选，待评估

Phase 6（个性化）
  ├─ 6.1 背景自定义增强    ← 进阶壁纸功能
  ├─ 6.2 快捷键系统        ← 效率提升
  ├─ 6.3 数据导出导入       ← 实用性高
  └─ 6.4 设置面板重构       ← UI 一致性
```

## 五、技术架构建议

### 新增模块
```
modules/
  ├── weather.js      # 天气数据获取 + 缓存
  ├── wallpaper.js    # 壁纸管理 + 存储
  ├── bookmarks.js    # 书签 CRUD + 存储
  ├── hitokoto.js     # 一言 API
  ├── zen.js          # 极简模式
  ├── settings.js     # 设置面板（统一入口）
  └── shortcuts.js    # 快捷键注册
```

### 存储结构（chrome.storage.local）
```js
{
  bookmarks: [...],        // 快捷书签数组
  wallpaper: {
    type: 'gradient',       // 'gradient' | 'image' | 'upload'
    value: '#xxx',          // 渐变色 / 图片URL
  },
  weather: {
    city: '上海',
    coords: { lat, lon },
    cached: { temp, condition, time }
  },
  settings: {
    showHitokoto: true,
    showHotSearch: false,
    theme: 'system',
    lang: 'zh'
  }
}
```

## 六、风险与约束

| 风险 | 应对 |
|------|------|
| 天气 API CORS 限制 | Open-Meteo 支持 CORS，或通过 background.js 转发 |
| 热搜 API 不稳定 | 提供开关，允许用户关闭此功能 |
| 壁纸本地存储体积 | 限制用户上传图片 < 2MB，压缩存储 |
| 国际化字符串膨胀 | i18n 模块已就绪，新增 key 即可 |
| 性能：壁纸 + 动画 | 确保 60fps，必要时降级 |
